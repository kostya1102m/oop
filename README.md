
В данной UML-Диаграмме класса Date представлены атрубуты класса, конструктор класса, а также два публичных метода: add и subtract; приватный метод - daysInMonth. Атрибуты year, month, day, hour, minute, second и IsOurEra являются приватными и могут быть доступны только внутри класса. Конструктор принимает значения для каждого из атрибутов объекта и использует методы set для установки этих значений. Методы set используются для проверки входных параметров и установки значений соответствующих атрибутов и являются защищённными.
Если входные параметры не проходят проверку, то атрибуты класса (поля обьекта) выставляются в значение по умолчанию и сообщает об ошибке.
Методы add и subtract возвращают новый объект класса Date, принимают на вход целочисленные параметры и производят добавление и вычитание "времени" из объекта класса соответственно.
Метод daysInMonth на основе проверки високосности года сравнивает значение `month` с различными значениями, чтобы определить, к какому из трех случаев относится месяц:
1. Если month равен 2, то это февраль. Затем проверяется, является ли год високосным.В случае, если год високосный, метод возвращает 29. В противном случае, метод возвращает 28.
2. Если month равен 4, 6, 9 или 11, то это апрель, июнь, сентябрь или ноябрь соответственно. Метод возвращает 30, так как эти месяцы всегда содержат 30 дней.
3. Во всех остальных случаях (январь, март, май, июль, август, октябрь, декабрь), метод возвращает 31, так как эти месяцы всегда содержат 31 день.
Также В классе Date перегружены следующие операторы: оператор копирования, <<, =, +, +=, -, -=, >, <, ==(не представлены в UML-Диаграмме).
